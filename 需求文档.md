# AList/OpenList 安卓原生客户端产品需求文档 (PRD)

## 1. 项目概述

* **产品定位**：一款专为 Android 用户打造的高效率、无套壳的纯原生 AList/OpenList 客户端。
* **技术选型**：Kotlin + Jetpack Compose + Material 3。
* **核心目标**：提供原生级（60/120帧）的流畅文件浏览体验，通过深度整合安卓底层文件系统，实现云端文件与本地存储的无缝互通。

---

## 2. 功能需求详情

### 2.1 服务器连接与账户管理（多配置文件）

* **多服务器管理**：
* 首页顶部提供**下拉菜单**，用于在不同的服务器配置文件之间进行一键切换。
* 支持配置文件的管理功能，包括：**创建新配置、删除、编辑、重命名**。


* **安全与证书管理**：
* 支持 **HTTPS** 和 **HTTP** 两种连接协议（以单选框/切换开关的形式供用户选择）。
* 针对自建服务器或自签证书，提供“忽略 SSL 证书错误”的全局开关。


* **凭据加密**：
* 用户的服务器密码、Token 等敏感凭据必须使用 Android 底层的 **Android Keystore** 进行安全加密存储，防止设备被 root 或脱库导致数据泄漏。


### 2.2 文件浏览器与基础操作

* **文件列表视图**：展示 AList 目录树，采用 Compose 的 `LazyColumn` 实现大列表的高性能流畅滚动。
* **排序与多维筛选**：
* **排序功能**：支持按名称、大小、修改时间进行正序/倒序排列。
* **文件夹置顶**：提供一个全局开关——**“文件夹始终置顶”**，开启后无论何种排序规则，文件夹均排在文件上方（文件夹遵循排序规则）。
* **多维度文件筛选**（更新）：支持以下两种过滤模式，协助用户在当前目录下快速定位文件：
1. **文件大类筛选**：提供快捷过滤标签（Tabs/Chips），支持按 **视频、图片、音频、文档、其他** 5 种核心类型进行一键分类筛选。
2. **精准后缀过滤**：支持用户手动输入或选择特定的**文件后缀名/扩展名**（如 `.mp4`、`.pdf`、`.zip`），对当前目录进行精确的自定义模糊/全匹配过滤。




* **基础常规操作 (CRUD)**：
* 支持：**新建文件夹、重命名、删除、复制、移动**文件或文件夹。


* **本地缓存与异步加载**：
* 引入本地数据库缓存已加载的目录结构，支持离线浏览，再次进入相同目录时秒开。


* **下拉刷新与分页设置**：
* 列表支持标准下拉刷新（Swipe-to-refresh）以同步服务端最新状态。
* 支持 API 分页懒加载。并在设置中提供“分页大小（Page Size）”设置项，允许用户自定义单次请求加载的文件数量（如 30/50/100 条）。



### 2.3 文件传输管理（上传与下载）

* **后台传输与前台服务**：
* 传输引擎基于 Android `WorkManager`（适合小文件或异步队列）与 `Foreground Service`（前台服务）构建。
* 在进行大文件传输时，在系统通知栏常驻显示传输进度条，确保应用退到后台或系统锁屏时不会被垃圾回收（杀后台）。


* **断点续传与线程控制**：
* 传输协议支持**断点续传**，在网络切换或断网重连后可自动或手动恢复未完成的任务。
* 设置中允许用户限制**最大并发传输线程数**。


* **传输列表可视化**：
* 提供独立的传输管理页面，采用 Tab 标签页清晰划分：**“正在传输”**、**“已完成”**、**“传输失败”**。



### 2.4 多媒体与文档视图

* **视频播放（调用第三方）**：
* **不集成**内置视频解码内核（如 ExoPlayer/VLC）。
* 用户点击视频时，通过 Android 原生 `Intent.createChooser` 弹窗，**调用系统内已安装的第三方专业播放器**（如 MX Player, VLC, PotPlayer 等）进行流媒体播放。


* **图片浏览器**：
* 集成纯原生图片查看器，支持手势缩放、双击放大、左右划动切换上一张/下一张图片。


* **PDF 与文档预览**：
* 集成主流的在线预览接口（如微软 Office Online 或谷歌文档查看器接口），实现不下载文件即可在 App 内直接预览 PDF、Word、Excel、PPT 等常见文档。



### 2.5 安卓系统级深度整合

* **接入 DocumentsProvider（核心体验）**：
* 继承并实现 Android 系统的 `DocumentsProvider`。
* 允许用户在打开系统自带的“文件（Files）”应用或其他支持 SAF 的第三方 App（如系统相册、WPS）时，直接在**系统侧边栏**看到本应用的 AList 虚拟盘，像访问本地文件夹一样直接读写云端文件。


* **调用系统分享（Share Sheet）**：
* **应用到系统**：在 App 内长按文件，可调出系统分享面板，将文件或下载链接发送至微信、QQ 等外部应用。
* **系统到应用**：在手机相册或其他文件管理器中选中文件点击分享时，可直接选择本 App，并弹窗让用户选择对应的 AList 路径进行一键上传。


* **适配 Scoped Storage（分区存储）**：
* 严格遵守现代 Android（Android 10-16+）的分区存储规范，下载文件默认保存至系统公共目录（如 Download/）或通过 SAF（存储访问框架）由用户授权的特定安全目录，无需申请侵入性的全盘读写权限。

## 3. 技术架构与非功能需求

### 3.1 性能与 UI 性能

* **全原生 UI 组件**：禁止使用 WebView 套壳，核心 UI 全部基于 **Jetpack Compose** 绘制，配合 **Material 3** 设计规范，确保 120Hz 刷新率下的丝滑体验。
* **文件图标规范**：
* **暂不考虑动态加载图片/视频的缩略图**。
* 文件列表统一使用系统或内置的**静态类型图标**（如文件夹图标、视频图标、图片图标、文档图标）。
* **技术预留**：在底层数据模型和网络请求层中**保留缩略图（Thumbnail）相关的接口与字段**，以便后续版本低成本升级扩展。



### 3.2 核心技术栈清单

* **网络层**：`Retrofit 2` + `OkHttp 4`（处理 AList 的 REST API，配置安全套接字以支持忽略 SSL 错误）。
* **数据持久层**：`Room` 数据库（用于多账号配置文件存储、传输历史记录、文件目录缓存）。
* **异步机制**：`Kotlin Coroutines（协程）` + `Flow`（处理文件流数据的异步加载，确保 UI 线程绝对不卡顿）。
* **图片加载**：`Glide` 或 `Coil`（用于图片预览时的内存优化，防止 OOM）。
